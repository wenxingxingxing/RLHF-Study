
# 输入`X`到 `Encoder Softmax` 前的完整流程

## 1. 输入与 Padding
输入 X：
一个 batch 内多个 prompt，长度不一，每个元素是 token id。

动态 Padding：
统计当前 batch 内最长序列长度 L = max(seq_len_raw)，所有短序列用 [PAD] token 补齐到 L。

得到 input_ids：
形状 (batch_size, L)，内容为 token id，其中 [PAD] 有固定的 id（如 0）。

## 2. 生成 Padding Mask

根据 input_ids 生成一个同等形状的 mask：

```python
mask = (input_ids != pad_token_id)  # 形状: (batch_size, L)
```

mask 含义：

- True（1）→ 真实 token，有效位置
- False（0）→ [PAD] token，需要被屏蔽的位置

## 3. 进入 Encoder → 计算 Attention Scores
输入经过 Embedding + 位置编码，得到 (batch_size, L, d_model)。

线性投影得到 Q, K, V，形状均为 (batch_size, num_heads, L, d_k)。

计算注意力分数：

```python
scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
```

scores 形状：(batch_size, num_heads, L, L)

含义：scores[b, h, i, j] 表示第 b 个样本、第 h 个头中，query 位置 i 对 key 位置 j 的原始注意力分数。

## 4. Mask 的形状调整与加法操作

scores 是 4D 张量，而 mask 是 2D 张量 (batch_size, L)，不能直接相加。需要将 mask 扩展到 key 维度，并广播到 scores 的形状。

操作步骤：

```python
# 1. 调整形状：在 num_heads 和 query 维度插入大小为 1 的维度
mask = mask.unsqueeze(1).unsqueeze(2)  # (batch_size, 1, 1, L)

# 2. 转换为加性掩码：有效位置=0，填充位置=-1e9
mask_additive = (1.0 - mask.float()) * -1e9

# 3. 应用到 scores 上（广播到 (batch_size, num_heads, L, L)）
masked_scores = scores + mask_additive
```

加法效果：

- 对于 有效 key 位置（mask=1）→ 加 0，分数不变
- 对于 填充 key 位置（mask=0）→ 加 -1e9，分数变成极小值

无论 query 是什么，**所有 query 对填充 key 的注意力分数都变成了 -1e9**，即填充位置在 key 维度上被完全屏蔽。

## 5. Softmax 与后续计算
```python
attn_weights = F.softmax(masked_scores, dim=-1)  # 形状: (batch_size, num_heads, L, L)
```

Softmax 后：

- 填充 key 位置的权重 ≈ 0
- 有效 key 位置的权重正常分布（和为 1）

```python
attn_output = attn_weights @ V  # (batch_size, num_heads, L, d_k)
```

填充位置对输出没有任何贡献，模型只关注真实 token。


## 6. 核心要点
- 输入 padding → 统一到 batch 内最大长度 L
- 生成 mask → 1 表示真 token，0 表示 pad
- Encoder 中 softmax 前 → 执行 masked_scores = scores + (1 - mask) * (-1e9)


# Decoder 完整流程

**Decoder 的核心任务是在训练时并行接收右移后的目标序列，通过自注意力中的因果掩码模拟自回归生成，同时通过交叉注意力融合编码器信息**

## 1. 输入准备：右移与 Padding
假设一个 batch 中目标序列（label）长度不一，我们对它进行两步处理。

**右移一位（Shift Right）**

- 在目标序列最前面插入 <SOS>（起始符号），去掉最后一个 token。
- 例如：目标 ["A","B","C","<EOS>"] → 右移后 ["<SOS>","A","B","C"]
- 这样得到的序列就是 Decoder 的输入（通常称为 tgt_input 或 decoder_input_ids）。

动态 Padding

- 统计当前 batch 内右移后序列的最大长度 L_tgt。
- 将所有序列用 [PAD] token 补齐到 L_tgt。
- 得到 decoder_input_ids：形状 (batch_size, L_tgt)。

对应的目标序列（label）也 pad 到同一长度 L_tgt，但其内容保持为原始目标序列右移前的内容，例如 ["A","B","C","<EOS>","PAD"]。注意：label 中 <SOS> 永远不出现，因为模型预测的是“下一个词”。

## 2. 生成两种 Mask
Decoder 的自注意力层需要同时使用两种 mask：padding mask 和 causal mask。


2.1 Padding Mask（填充掩码）
- 来源：decoder_input_ids 中不等于 [PAD] 的位置为 1，等于 [PAD] 的位置为 0。
- 形状：(batch_size, L_tgt)，1 表示真实 token（包括 <SOS>），0 表示 pad。
- 注意：<SOS> 也是真实 token，因为它需要作为前文信息参与注意力，所以 mask 中置 1。

2.2 Causal Mask（因果掩码/未来掩码）
- 目的：保证位置 i 只能看到 ≤ i 的 token，不能看到未来。
- 形状：(L_tgt, L_tgt) 的下三角矩阵（包括对角线），允许位置为 0，屏蔽位置为 -∞。
- 通常扩展为 (1, 1, L_tgt, L_tgt) 以便广播。

矩阵示意（L_tgt=4，0 表示允许，-∞ 表示屏蔽）：

```python
        j=0  j=1  j=2  j=3
i=0:     0  -∞  -∞  -∞
i=1:     0   0  -∞  -∞
i=2:     0   0   0  -∞
i=3:     0   0   0   0

```

## 3. Decoder 第一层：Masked Multi-Head Self-Attention

3.1 线性投影，计算 Scores
输入 decoder_input_ids 经过 Embedding + 位置编码，得到 (batch_size, L_tgt, d_model)。
线性投影得到 Q, K, V，形状均为 (batch_size, num_heads, L_tgt, d_k)。

```python
scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
# 形状: (batch_size, num_heads, L_tgt, L_tgt)
```

3.2 合并两种 Mask 并加到 Scores 上

```python
# 处理 Padding Mask
pad_mask = (decoder_input_ids != pad_token_id)          # (batch_size, L_tgt)
pad_mask = pad_mask.unsqueeze(1).unsqueeze(2)           # (batch_size, 1, 1, L_tgt)
pad_mask_additive = (1.0 - pad_mask.float()) * -1e9     # 有效位置=0, pad位置=-1e9

# 处理 Causal Mask
causal_mask = torch.triu(torch.ones(L_tgt, L_tgt), diagonal=1) * -1e9   # 上三角为-1e9
causal_mask = causal_mask.unsqueeze(0).unsqueeze(0)                     # (1, 1, L_tgt, L_tgt)

# 合并（两者相加）
combined_mask = pad_mask_additive + causal_mask
# 形状: (batch_size, 1, 1, L_tgt) + (1, 1, L_tgt, L_tgt) -> 广播为 (batch_size, num_heads, L_tgt, L_tgt)

# 应用到scores

masked_scores = scores + combined_mask
attn_weights = F.softmax(masked_scores, dim=-1)
attn_output = attn_weights @ V


```
效果：

- 每个 query 对所有未来 key 的注意力分数为 -∞，softmax 后为 0。
- 每个 query 对所有 pad key 的注意力分数也为 -∞，softmax 后为 0。
- 只有 j ≤ i 且 key 不是 pad 的位置才允许注意力。


## 4. Decoder 第二层：Cross-Attention
在 Cross-Attention 中：

- Q 来自 Decoder 上一层的输出（经过自注意力后），形状 (batch_size, num_heads, L_tgt, d_k)。
- K 和 V 来自 Encoder 的最终输出，形状 (batch_size, num_heads, L_enc, d_k)。
需要的 Mask：只有 Encoder 的 Padding Mask，因为解码器可以关注编码器输出中的所有有效位置，没有因果限制。

操作：

```python
enc_pad_mask = (encoder_input_ids != pad_token_id)      # (batch_size, L_enc)
enc_pad_mask = enc_pad_mask.unsqueeze(1).unsqueeze(2)   # (batch_size, 1, 1, L_enc)
enc_pad_mask_additive = (1.0 - enc_pad_mask.float()) * -1e9

scores_cross = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
# 形状: (batch_size, num_heads, L_tgt, L_enc)

masked_scores_cross = scores_cross + enc_pad_mask_additive
attn_weights_cross = F.softmax(masked_scores_cross, dim=-1)
attn_output_cross = attn_weights_cross @ V
```

效果：解码器的每个位置都能关注编码器输出的所有有效 token，但忽略其中的 [PAD] 位置。

## 5. 输出与损失计算
Decoder 堆叠多层（每层依次执行 Masked Self-Attention 和 Cross-Attention）后，经过最后的线性层和 softmax，得到预测 logits：

```python
logits: (batch_size, L_tgt, vocab_size)
```

损失计算：

- 标签（label）是右移前的目标序列，同样 pad 到 L_tgt，形状 (batch_size, L_tgt)。
- 使用交叉熵损失，通常设置 ignore_index=pad_token_id，让模型不计算 pad 位置的损失。
- 这样，模型只在真实 token 的位置上学习预测下一个词。

<font color=yebl>Decoder 既通过 Causal Mask 保证了**自回归生成的因果性**，又通过 Padding Mask 消除了**填充噪声**，同时利用 Cross-Attention 提取编码器的上下文信息，完整实现了序列到序列的生成任务。</font>
